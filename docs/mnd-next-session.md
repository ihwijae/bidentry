# 국방부 참가신청 자동화 인계 (2025-11-27)

## 미해결 문제 요약
1. **참가신청 단계의 공인인증서 선택 오류**
   - 참가신청서 마지막 `[신청]`을 누른 뒤 뜨는 인증서 창에서, 로그인 시 사용했던 회사의 인증서를 찾지 않고 목록 첫 번째 행을 선택함.
   - 결과적으로 `Certificate confirm failed: 갱신 안내`와 같이 대상 인증서가 아닌 다른 인증서를 시도하면서 실패.
   - 로그 예시: `2025-23219 참가신청 실패: [MND] 참가신청 인증서 처리 실패: Certificate confirm failed: 갱신 안내` (engine_runs: 2025-11-27T08-54-04 run)
   - 현재 로그인 단계에서 선택했던 인증서 정보(`selectionHint`)를 전달하도록 수정했지만, 참가신청 단계에서 정확히 일치하는 인증서를 찾지 못하고 있음. `handleNxCertificate`는 hint 기반 점수를 부여하도록 되어 있으므로, 실제 DOM에서 hint와 일치하는 셀 텍스트를 추가로 확인하거나, 인증서 리스트의 구조를 다시 분석할 필요가 있음.

2. **사후심사대상 입찰안내 팝업 검증 미완료**
   - 참가신청 후 새로운 공고로 넘어가기 전에 `사후심사대상 입찰안내` 팝업을 닫아야 함.
   - 팝업 닫기 로직을 보강했지만, 문제 1로 인해 다음 단계로 진행하지 못해서 실제로 정상 닫히는지 최종 확인하지 못함.

## 다음 작업 제안
- 참가신청 단계 인증서 창에서 실제로 렌더링되는 인증서 리스트의 텍스트/셀 구조를 `engine_runs` 덤프나 스크린샷으로 확보하고, `selectionHint`와 일치하도록 가중치를 조정하거나, 확정적인 셀 비교 로직을 추가.
- 인증서 선택이 통과한 뒤, 문제 2의 팝업 닫기 동작을 재검증하고 필요 시 추가 덤프(`post_review_popup_forced`, `post_review_popup_window`)를 활용해 셀렉터를 다듬기.
