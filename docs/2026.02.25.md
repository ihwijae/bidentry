# 2026.02.25

## 목적
- KEPCO 입찰참가신청 자동화 안정화
- 잘못된 버튼 클릭/팝업 닫기 오동작 제거
- 참가신청 플로우 순서 고정

## 변경 요약
- 브라우저 기본값을 Chrome으로 유지(Edge 이슈 회피)
- KEPCO 공고번호 입력 오탐(날짜 필드 선택) 수정
- `입찰참가신청` 버튼 탐색/클릭 로직 재구성
  - 동일 텍스트(상단/좌측/탭)와 충돌하지 않도록 그리드 툴바 기준으로 한정
  - `btn-request` 우선 타겟팅 + 실제 버튼 컴포넌트 클릭
- `입찰참가신청` 클릭 후 확인창 `예` 자동 클릭 추가(id 비고정)
- `예` 다음 체크박스(`role=checkbox`, x-form-checkbox) 자동 클릭 추가
- 안내창 닫기 로직을 최종 플로우에 맞게 조정
  - 메인 진행 모달(체크박스/최종제출 단계)은 보호
  - 다음 화면 진입 확인 후 안내창 정리 실행
- 최종 화면 체크박스 일괄 체크 + 최종 제출 클릭 흐름 정리
- 실패 시 HTML dump/에러 전파 강화(실패를 성공으로 오인하지 않도록 수정)
- BOM 제거 및 깨진 문자열로 인한 문법 오류 정리

## 상세 변경 내용
### 1) 공고번호 입력 안정화
- 입력 필드 후보 점수화로 공고번호 필드 우선 선택
- `공고일자/신청마감/date/from/to` 계열 필드 제외
- 입력 검증 실패 시 `bid_input_failed` HTML dump 저장

### 2) 입찰참가신청 버튼 클릭 안정화
- 버튼 텍스트 단순 매칭 제거
- 실제 액션 버튼(`grid-toolbar` + `btn-request`) 우선 탐색
- 클릭 타겟 id 로깅: `apply target id=...`

### 3) 모달/팝업 처리 순서 정리
- 확인창 `예` 클릭
- 확인 체크박스 클릭
- 최종 화면 진입 여부 확인(`final stage detected=...`)
- 이후 안내창 정리 실행
- 최종 화면 체크박스들 체크 후 제출

### 4) 실패 처리 강화
- 버튼 미탐색/후속 화면 미감지/최종 제출 미확인 시 즉시 실패 처리
- 상태 보고(`bid_status`)가 실제 결과와 일치하도록 정리
