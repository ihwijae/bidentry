# 2026-02-27 작업 정리 (KEPCO 참가신청 자동화)

## 1) 목표
- 한전 참가신청 최종 단계 자동화를 안정화
- 공고번호 다건 처리 시 공고별 상태(`start/done/error`)를 정확히 표시
- 실패 시 즉시 중단(다음 공고 미진행) 정책 유지

## 2) 오늘 구현/보완한 핵심

### 2.1 최종 제출 단계 안정화
- 최종 화면 체크박스 자동 선택 로직 보완
- 제출 버튼 탐색을 동적 텍스트 기반(`제출`)으로 처리
- 제출 후 `예/아니오` 확인창 반복 처리(인증서 창 감지 시 반복 중단)
- 제출 후 공인인증서 모달 감지/처리 연동
- 완료 팝업(`제출/완료`)의 `확인` 버튼 클릭 후 닫힘 확인을 완료 신호로 사용

### 2.2 팝업 처리 개선
- 차단성 `알림/안내` 팝업 자동 닫기 보강
- 워크플로우 필수 창(체크 필요한 창) 보호
- 팝업 외 본문/탭의 `닫기`를 누르지 않도록 범위 제한

### 2.3 체크박스 처리 개선
- 이중 클릭(체크 후 즉시 해제) 문제 제거
- Ext 체크박스(`role=checkbox`, `input type=button`) 대응 보강
  - DOM 클릭 실패 시 `Ext.getCmp(...).setValue(true)` fallback
  - 상태 판정 시 `checked` + `aria/class` + Ext `getValue()` 병행
- 공고 단위 상태 초기화 추가
  - 이전 공고 체크 상태 캐시가 다음 공고에 누적되지 않도록 초기화

### 2.4 성공/실패 판정 보정
- `이미 참가신청 불가` 문구 감지 시 성공 처리하지 않고 `error`로 처리
- 제출 감지 누락 대비 지연 성공 판정(`late completion probe`) 추가
  - 단, 실제 완료 신호(완료 팝업 닫힘 등) 있을 때만 성공 전환

### 2.5 다건 처리 정책 정렬
- KEPCO도 MND와 동일하게 "중간 실패 시 즉시 중단"으로 정렬
- 공고 1~N 처리 중
  - 성공 건: `done`
  - 실패 건: `error`
  - 실패 발생 시 해당 지점에서 전체 작업 종료

## 3) 오늘 발생했던 주요 이슈와 원인
- 스크롤/체크 후 제출 실패: 체크 상태 판정 불일치(Ext 체크박스)
- 팝업 닫기 후 화면 이탈: 팝업 외 `닫기`까지 누른 부작용
- 성공했는데 실패 표시: 제출 클릭 감지 누락과 완료 판정 시점 불일치
- 이미 신청 공고가 성공 표시: `alreadyApplied` 분기 설계 문제

## 4) 현재 최종 동작 기준
- 최종 화면 진입 -> 체크박스 동적 처리 -> 미체크 검증 통과 시만 제출
- 제출 후 확인창 `예` 반복 처리 -> 인증서 모달 처리
- 완료 팝업 `확인` 클릭/닫힘 확인 시 공고 완료 처리
- 공고별 상태 이벤트 emit
- 중간 실패 시 즉시 중단

## 5) 확인된 현재 상태
- 공고 2건 연속 테스트에서 전체 흐름 완료(`done`) 확인
- 체크박스 처리 방식이 클릭 중심에서 Ext 값 설정 중심으로 바뀌어
  - 스크롤 움직임이 덜 보일 수 있으나 정상 동작 가능
