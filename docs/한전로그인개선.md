# 한전 로그인 개선

## 배경
- 증상: KEPCO 로그인에서 ID/PW 창은 빠르게 보이지만, 입력 시작까지 대기 시간이 길게 발생.
- 관찰: 타임스탬프 로그(`[KEPCO][TS]`) 기준으로 병목은 ID/PW 입력 로직 자체보다 로그인 링크 클릭 전/직후 구간에 집중.

## 핵심 분석
- 기존 로그에서 `loginLink:click:end` 시점이 이미 큰 누적 시간(예: +91초대)을 차지.
- 반면 ID/PW 입력 관련 구간은 상대적으로 짧음(필드 획득/입력/검증 합산 수초 내).
- 결론: "입력이 느린 것처럼 보이는 문제"의 주원인은 로그인 링크 탐색/클릭 대기 누적.

## 적용한 개선 사항

### 1) 로그인 링크 클릭 로직 단순화
- 변경 전:
  - 후보 selector를 순차 탐색하면서 클릭/팝업 대기를 반복할 수 있는 구조.
- 변경 후:
  - 로그인 트리거를 먼저 1개 확정하고 1회 클릭.
  - 팝업 대기는 짧게 유지하고, 팝업이 없으면 즉시 다음 단계 진행.

### 2) 로그인 링크 탐색 경량화
- 변경 전:
  - poll 루프에서 메인/프레임 탐색 비용이 누적될 수 있음.
- 변경 후:
  - 메인 컨텍스트 중심으로 빠르게 탐색해 루프 비용 감소.

### 3) ID/PW 필드 입력 fast-path
- 변경 전:
  - 보수적인 다중 selector/다중 컨텍스트 탐색을 먼저 수행.
- 변경 후:
  - `#username/#password`(및 대표 쌍)를 우선 확인해 즉시 입력.
  - 실패 시에만 기존 확장 탐색(라벨/프레임/재시도) 수행.

### 4) 제출 후 대기 최적화
- 변경 전:
  - 제출 후 `waitForNavigation` 타임아웃 대기 비중이 큼.
- 변경 후:
  - `짧은 sleep / navigation / popup` 중 먼저 완료되는 조건으로 빠르게 탈출.

### 5) 타임스탬프 계측 추가
- `[KEPCO][TS]` 로그를 단계별로 추가하여 병목 구간을 ms 단위로 확인 가능.
- 주요 마커:
  - `loginLink:wait:start/end`
  - `loginLink:click:start/end`
  - `fields:acquire:start/end`
  - `fields:input:start/end`
  - `fields:verify:start/end`
  - `submit:click|enter:start/end`

## 관련 파일
- `automation-engine/src/sites/kepco.js`
- `automation-engine/src/web/playwright.js` (옵션 전달)

## 결과 요약
- 체감상 느린 구간의 원인을 ID/PW 입력 자체가 아닌 "로그인 링크 구간"으로 특정.
- 해당 구간 중심으로 로직을 단순/경량화하여 로그인 전체 체감 속도 개선.
- 향후 동일 증상 재발 시 `[KEPCO][TS]` 로그로 즉시 병목 재진단 가능.

## 참고
- `kepco.js`의 BOM(Byte Order Mark)은 제거 완료(동작 영향 없음, 파일 정리 목적).
